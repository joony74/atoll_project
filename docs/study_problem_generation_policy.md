# Study Problem Generation Policy

## 목적

코코앱의 학습리스트 채팅에서 사용자가 `다른문제`, `비슷한 문제`, `한 문제 더`처럼 요청했을 때, 일반 LLM 대화로 넘기지 않고 현재 학습 카드와 같은 개념의 변형 문제를 제공한다.

이 기능의 목적은 단순한 랜덤 문제 출제가 아니라, 현재 등록된 문제를 기준으로 학생이 바로 이어서 연습할 수 있는 유사 문제를 만드는 것이다.

## 현재 생성 기준

현재 구현은 선택된 학습 카드의 분석 결과를 기준으로 한다.

1. 현재 선택된 문서의 `analysis.structured_problem.metadata.visual_template.rule_id`를 확인한다.
2. 해당 `rule_id`에 대응하는 로컬 템플릿 생성기를 선택한다.
3. 원래 문제의 숫자, 풀이 구조, 답 형식을 읽는다.
4. 같은 풀이 전략이 필요한 다른 숫자 조합으로 변형한다.
5. 생성 문제의 정답은 저장된 값이 아니라 내부 계산으로 다시 만든다.
6. 생성 문제 텍스트를 바탕으로 간단한 문제 그림 PNG를 만들어 답변 카드 오른쪽에 붙인다.
7. 생성한 유사 문제는 문서별 임시 상태에 저장한다.
8. 이어서 사용자가 `풀이해줘`라고 입력하면 원래 이미지 문제가 아니라 방금 생성한 유사 문제를 풀이한다.
9. 생성 문제의 풀이가 완료되기 전에는 다음 `다른문제` 요청을 막는다.
10. 한 학습 카드당 최대 10개의 유사 문제를 제공하고, 10개를 모두 풀면 더 이상 제출하지 않는다.

## 예시

원래 문제:

```text
9와 6을 10을 이용하여 모으고 가르세요.
```

원래 풀이 기준:

```text
9 + 6 = 15
15 = 10 + 5
정답: 빈칸: 15, 5
```

`다른문제` 요청 시 생성 예시:

```text
좋아요. 같은 풀이 기준으로 비슷한 문제를 하나 내볼게요.

문제: 9와 5를 10을 이용하여 모으고 가르세요.

힌트: 두 수를 먼저 모은 다음, 그 수를 10과 나머지로 가르면 됩니다.
풀이가 필요하면 `풀이해줘`라고 해주세요.
```

이후 `풀이해줘` 요청 시:

```text
9와 5를 모으면 9 + 5 = 14입니다.
14는 10과 4로 가를 수 있습니다.
정답: 빈칸: 14, 4
```

풀이하지 않은 상태에서 다시 `다른문제`를 요청하면:

```text
먼저 제출된 문제 풀이를 완료하여야 다른 문제가 제출됩니다.
```

10개를 모두 풀고 다시 `다른문제`를 요청하면:

```text
해당 학습문제를 전부 풀었습니다.
```

## 현재 지원 템플릿

현재는 초등 시각 연산 템플릿을 우선 지원한다.

- `generic_make_ten_compose_decompose`
  - 두 수를 10 기준으로 모으고 가르기
  - 예: `9와 6을 10을 이용하여 모으고 가르세요.`
- `generic_make_ten_addition_decomposition`
  - 10 만들기 덧셈
  - 예: `8+5에서 5를 2와 3으로 가르고 10을 만들어 계산하세요.`
- `generic_make_ten_subtraction_decomposition`
  - 10 이용 받아내림 뺄셈
  - 예: `14-6에서 6을 4와 2로 가르고 10을 이용해 계산하세요.`

지원되지 않는 템플릿의 경우 현재는 유사 문제 생성을 하지 않고 기존 채팅 경로로 넘긴다.

## 구현 위치

주요 구현은 다음 파일에 있다.

- `app/chat/study_fast_reply.py`
  - `다른문제` 계열 프롬프트 감지
  - 현재 문제의 `rule_id` 기반 유사 문제 생성
  - 문서별 10문제 생성 세션 관리
  - 생성 문제별 풀이 완료 여부 관리
  - 이어지는 `풀이해줘` 요청을 생성 문제 풀이로 연결
- `app/chat/practice_image.py`
  - 생성 문제 텍스트와 템플릿 ID를 문제 그림 PNG로 렌더링
  - 생성된 이미지는 채팅 카드 오른쪽 미리보기로 표시
- `app.py`
  - 학습리스트 후속 질문에서 빠른 로컬 응답을 LLM보다 먼저 호출
  - 로컬 풀이 응답도 약 3초 로딩 후 표시하도록 처리

## LLM을 바로 사용하지 않는 이유

`다른문제`를 LLM에 바로 맡기면 다음 문제가 생긴다.

- 현재 문제 유형과 다른 문제를 낼 수 있다.
- 방금 풀이한 전략과 다른 풀이법이 필요한 문제를 낼 수 있다.
- 정답 계산과 풀이가 일관되지 않을 수 있다.
- 초등 저학년 문제에서 표현이 너무 어렵거나 교과서 문체와 달라질 수 있다.
- 메모리와 응답 시간이 증가한다.

따라서 현재 단계에서는 검증 가능한 로컬 템플릿 생성을 우선 사용한다.

## 현재 한계

현재 방식은 문제은행 전체에서 문제를 고르는 방식이 아니다. 즉, 아직은 다음 기준을 모두 반영하지 않는다.

- 학년별 전체 커리큘럼
- 단원별 출제 비율
- 난이도 태그
- 학생의 이전 오답 이력
- PDF 원본 교재의 문항 표현 다양성
- 그래프, 표, 도형, 문장제의 폭넓은 변형

현재는 “현재 문제와 같은 구조의 로컬 변형 문제”를 안정적으로 만드는 단계다.

## 다음 확장 방향

다음 단계에서는 문제 생성 기준을 아래 순서로 확장한다.

1. 현재 문제의 `school_level`, `grade`, `semester`, `unit`, `topic`, `concept_tags`를 함께 사용한다.
2. 문제은행에서 같은 개념과 유사 난이도의 문제를 먼저 검색한다.
3. 문제은행에 적절한 문제가 있으면 그 문제를 우선 제시한다.
4. 문제은행 후보가 부족하면 로컬 템플릿 생성기를 사용한다.
5. 생성 문제에는 다음 메타데이터를 남긴다.
   - 원본 문서 ID
   - 원본 `rule_id`
   - 생성 템플릿 ID
   - 생성 숫자 파라미터
   - 계산된 정답
   - 난이도
   - 생성 시각
6. 학생이 `풀이해줘`, `정답`, `힌트`를 요청하면 생성 문제 상태를 기준으로 응답한다.

## 목표 상태

최종 목표는 다음 흐름이다.

```text
이미지/PDF 등록
-> 문제 인식
-> 정규화된 문제 구조 생성
-> 풀이 및 정답 검증
-> 같은 개념의 유사 문제 추천 또는 생성
-> 생성 문제 풀이
-> 학생 반응을 다시 학습엔진에 반영
```

이 흐름이 안정화되면 코코엔진은 단순 풀이 도구가 아니라, 학생이 방금 풀던 문제를 기준으로 맞춤형 연습 문제를 이어서 제공하는 학습 엔진이 된다.
